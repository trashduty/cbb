# Spreads Analysis - REGRESSED (Lookup Tables)

## Methodology

### Step 1: Calculate Combo Spread
- For each model combination, compute the mean of the selected model spreads
- Example: KP+BT+EM combo_spread = (spread_kenpom + spread_barttorvik + spread_evanmiya) / 3
- Round to nearest 0.5 for lookup

### Step 2: Look Up Probability
- Use `spreads_lookup_combined.csv` with inputs:
  - `total_category`: 1 (total ≤137.5), 2 (137.5-145.5), 3 (>145.5)
  - `market_spread`: Opening spread line (rounded to 0.5)
  - `model_spread`: Combo spread from Step 1
- Returns `cover_prob`

### Step 3: Calculate Edge (REGRESSED)
- regressed_spread = 0.6 × market_spread + 0.4 × combo_spread
- Look up `cover_prob` using `regressed_spread` (not raw combo spread)
- Edge = cover_prob - 52.38%
- 52.38% is break-even at -110 odds
- Regression pulls the model forecast toward the market line before lookup

### Step 4: Bet Direction
- If edge > 0: Bet COVER (model says team covers the spread)
- If edge < 0: Bet AGAINST (model says team does NOT cover)
- Filter: Only bet when |edge| >= threshold

### Step 5: Calculate ROI
- Standard -110 juice: Win pays 100/110, loss costs 1
- ROI = ((wins × 0.909) - losses) / total_bets × 100

### Filters Applied
- Home team rows only (one bet per game)
- Games must have: opening_spread, opening_total, spread_covered
- All models in the combination must have data for that game
